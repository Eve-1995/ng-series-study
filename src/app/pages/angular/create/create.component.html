初识Angular CLI

CLI是 命令行界面(command-line interface)的缩写, 而 Angular CLI 是一个Angular官方出品的工具.这里要引用 大漠老师曾经打过的一个比方, 他说 Angular CLI是一辆坦克, 帮助你直接开战.我觉得这个比方是很合适的.因为在现代前端工程当中, 一个 SPA项目, 普遍需要scss, less, webpack, e2e, unit test等等复杂的技术, 如果你是初涉现代前端工程, 是不是感觉都没听过?
我当年只知道一个网页是由HTML, CSS, JS组成的...充其量知道点H5标签.
问题不大!在 Angular CLI这辆超级坦克面前, 开发现代前端工程的门槛变得不再那么遥不可及, 他会帮我们自动建立这些黑科技的配置并且应用在项目中, 现在我们开始建立属于自己的第一个Angular项目.

建立开发环境

Angular需要 Node.js (至少8.x)和Npm(5.x)的环境, 直接安装Node.js就会自动安装Npm了.这里推荐大家再安装一个 yarn, 功能与npm差不多, 但是我喜欢用这个软件安装第三方依赖.具体差异可以自行百度.
完全没听过npm的朋友看下这段话(老手请略过).非常理解初学者看到这些名词一脸懵逼, 让我来解释下npm是什么.
请尝试回忆下平时你是如何管理第三方依赖的?(如bootstrap, jquery, 这些应该不会没听过吧...), 我当年的做法是建立如下的目录结构:
project
    html 
        index.html 
    css 
        index.css 
    js 
        index.js 
    lib 
        bootstrap 
            bootstrap.css 
            bootstrap.js 
        jquery 
            jquery.js 
大家普遍都是这么做的吧, 把自己的js和css放在各自的文件夹里, 而第三方依赖统一放在一个文件夹下面, 那么请问, 你为什么要把第三方依赖放在统一文件夹下呢?一般是为了方便管理对吗?那如果你这个项目有很多人一起工作, 那你是不是需要列一个清单告诉别人, 需要哪些依赖, 版本分别是什么呢?显然这样的过程很麻烦, npm就应运而生了.现代前端工程中, 建立完项目后会有一个 package.json文件, 里面就包含了哪些依赖及其版本信息, 总之, 他就是用来管理我们的第三方依赖的.

创建新项目

使用以下命令来创建新项目, new表示要创建一个新项目, first-app表示项目的名字
ng new first-app
在这之后, Angular CLI会自动创建相应的文件, 使用npm安装依赖, 安装的文件都在, 这里需要花费不少时间.快的几分钟, 慢的就不一定了, 一般如果很久很久(比如半小时)都没结束, 可以尝试按下 ctrl+c停止任务, 使用 yarn进行安装, 输入如下命令即可.
yarn
npm的服务器在国外, 很多时候访问会有问题.因此推荐使用 cnpm, 或者yarn(推荐).

启动开发服务器

以往的网页双击打开即可.但是现代前端工程一般都需要本地服务器, 至于原因, 如果你是初学者很难理解, 所以就先记住即可.
输入如下命令启动开发服务器
ng serve --open
--open是可选参数, 其实可以不输入的, 这是为了让开发服务器启动完毕后, 自动使用浏览器打开项目页面, 默认为4200端口.在执行完命令后会出现一片绿色的信息, 就意味着服务器开启成功了.
作为一名开发者, 你一定会喜欢上这个颜色的, 希望每一天都是这么的绿...
